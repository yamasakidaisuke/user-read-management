# DESIGN.md — UI デザインガイド

v1.9「プロフェッショナル・フラット」。UI を生成・修正する前に必ずこのファイルを読むこと。

---

## 1. デザイン思考（新規 UI を作る前に必ず考えること）

### コンテキスト把握
- **目的**: 動物病院スタッフの投稿既読管理。データ密度の高い業務ツール。
- **ユーザー**: 獣医師・看護師・事務スタッフ。ITリテラシーは多様。迷いなく使える明快さが最優先。
- **トーン**: 「Industrial Utilitarian（産業的実用主義）」— 装飾を排し、情報の視認性と操作性に全振りする。

### 美的方向性
このプラグインは**意図的にミニマル**である。ミニマルは「手抜き」ではなく「精密な抑制」。
- 余白・整列・コントラスト・タイポグラフィの 4 要素だけで品質を担保する
- 「何を足すか」ではなく「何を削るか」で判断する
- 汎用的な AI 生成風デザイン（紫グラデ・過剰な角丸・意味のないアニメーション）は絶対に避ける

## 2. カラーパレット（厳守）

| 用途 | Hex | 設計意図 |
|---|---|---|
| ヘッダ背景 | `#2c3e50` | ダークスレート。権威性と落ち着き |
| ヘッダ下境界 | `#1a252f` | 深い陰影で区切りを明確化 |
| ヘッダセル境界 | `#34495e` | ヘッダ内の微細な区切り |
| 奇数行 | `#f8f9fa` | ほぼ白。ゼブラの淡い方 |
| 偶数行 | `#ffffff` | 純白。ゼブラの明るい方 |
| 行 hover | `#e3f2fd` | 寒色系の薄青。現在行の認識 |
| セル境界 | `#e0e0e0` | ニュートラルグレー |
| 編集可セル hover | `#fff3cd` | 暖色系の薄黄。「操作できる」のシグナル |
| 既読（済） | `#28a745` | 緑。完了・安全のセマンティクス |
| 未読（未） | `#dc3545` | 赤。要注意のセマンティクス |
| スクロール枠 | `#dee2e6` | 境界線。主張しすぎない |

**色の原則**: 支配色（ダークヘッダ）＋ 中性面（白/灰の本体）＋ 意味のあるアクセント（緑/赤）。色は情報を運ぶ手段であり、装飾ではない。

## 3. タイポグラフィ

- font-family は**指定しない**（WordPress テーマのシステムフォントスタックに委ねる）
  - 理由: 管理画面プラグインとしてテーマとの調和を優先。独自フォント読み込みによるパフォーマンス劣化を回避
- 階層はサイズと weight の組み合わせだけで構築する:

| 要素 | size | weight | padding |
|---|---|---|---|
| セクション見出し h2 | 20px | 600 | `10px 16px` |
| テーブルヘッダ th | 14px | 600 | `15px 12px` |
| テーブルセル td | 14px | 400 | `12px` |
| 先頭列（投稿タイトル） | 14px | 500 | — |
| ステータス文字（済/未） | 14px | 600 | — |

**原則**: サイズは 14px と 20px の 2 段階のみ。weight で階層を出す。

## 4. 空間構成

- セクション間: `margin-bottom: 40px` — 十分な呼吸を確保
- セクション見出し: `padding: 10px 16px` / 背景 `#2c3e50` / 文字白 / 下境界 `3px solid #1a252f`
- CSS 名前空間: `.p-read-status-*`（BEM 風。新規要素もこの規則に従う）
- CSS は `user-read-management.php` のショートコード出力内にインライン記述（別ファイル化しない）

## 5. テーブル構造（壊さないこと）

```html
<div class="p-read-status-section">
  <h2 class="p-read-status-section-title">診療マニュアル</h2>
  <div class="p-read-status-scroll-container">
    <table class="p-read-status-table">
      <thead><tr><th>…</th></tr></thead>
      <tbody><tr><td>…</td></tr></tbody>
    </table>
  </div>
</div>
```

### sticky 実装の必須条件（過去に何度も壊れた）
- `<thead>` / `<tbody>` タグは**必須**（省略すると sticky 不可）
- table: `border-collapse: separate; border-spacing: 0;`（collapse にすると sticky 境界が消える）
- table に `overflow: hidden` を**付けない**（v1.8 で撤去。sticky を殺す）
- scroll-container: `overflow: auto; max-height: 900px; position: relative;`
- `thead th` → `position: sticky; top: 0; z-index: 10;`
- 先頭列 `th:first-child, td:first-child` → `position: sticky; left: 0; z-index: 5;`
- 左上角 `thead th:first-child` → `z-index: 15;`

## 6. インタラクション

### Hover（色変化のみ。transform/shadow/scale は禁止）
- 行 hover: `background-color: #e3f2fd !important`
- 編集可セル hover: `background-color: #fff3cd` + `font-weight: bold`（「触れる」フィードバック）
- 読み取り専用セル `.p-read-status-cell-readonly`: hover 変化なし（操作不可を暗示）

### ステータス表示
- 済 → `color: #28a745; font-weight: 600`
- 未 → `color: #dc3545; font-weight: 600`
- **アクセシビリティ**: 色だけに依存せず、テキスト（済／未）で必ず識別可能にする

### フィードバック
- 操作成功/失敗は `alert()` またはボタンテキスト変更で伝達（トースト等の新規コンポーネントは導入しない）
- 確認ダイアログは `confirm()` を使用（カスタムモーダルは導入しない）

## 7. 日本語 UI 文言（変更禁止の固定文字列）

| 場面 | 文言 |
|---|---|
| チェックボックス | `読みました` |
| 状態 | `済` / `未` |
| セクション名 | `診療マニュアル` / `医療情報` / `必読資料` |
| CSV ヘッダ | `カテゴリー` / `投稿タイトル` |
| ボタン | `CSVエクスポート` / `エクスポート中...` |
| 確認 | `このステータスを [済/未] に変更しますか？` |
| 成功 | `CSVファイルをダウンロードしました` |
| 失敗 | `保存に失敗しました:` / `更新に失敗しました:` / `エクスポートに失敗しました:` / `AJAX エラーが発生しました` / `権限がありません` |

## 8. モバイル

- メディアクエリなし。`overflow: auto` による横スクロールで対応
- 独自レスポンシブ対応は追加しない（現仕様を維持）

## 9. 禁止事項（チェックリスト）

- [ ] グラデーション・ドロップシャドウ・角丸 (`border-radius`) を追加していないか
- [ ] アニメーション (`transition`, `animation`, `transform`) を追加していないか
- [ ] インライン CSS を別ファイルに分離していないか
- [ ] 絵文字を UI テキストに含めていないか
- [ ] 文言を英語化・変更していないか
- [ ] ロール権限周りの UI を変更していないか
- [ ] 外部フォントを読み込んでいないか
- [ ] 汎用 AI 風デザイン（紫グラデ、過剰な丸み、無意味な装飾）に陥っていないか
- [ ] sticky の前提条件（§5）を壊していないか
- [ ] 新規 CSS ブラウザ API に依存する変更を、Safari/Edge 未検証のまま入れていないか
